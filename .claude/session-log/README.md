# Session log

Эти файлы пишет Claude Code в конце каждого шага. Планирующий Claude
в чате читает их через GitHub. Не редактировать вручную задним числом —
если что-то выяснилось, новый файл с темой `<NN>-<MM>-followup-...`.

## Когда создавать файл

После каждого шага Stage N (см. `docs/08-roadmap.md`) — один файл.
Если шаг разделился на несколько коммитных порций, лог пишется один
суммарный в конце шага. Если шаг затянулся и часть осталась на
следующую сессию, файл всё равно создаётся со статусом `⚠️ частично`
и списком того, что не сделано.

## Имя файла

`<NN>-<MM>-<short-slug>.md`

- `NN` — двузначный номер Stage (`01`, `02`, ...).
- `MM` — двузначный номер шага внутри Stage (`01`, `02`, ...). Если
  шаг получился составным, можно слитное имя
  `01-01-02-alembic-and-engine.md`.
- `<short-slug>` — kebab-case, ASCII, без пробелов, чтобы grep'алось
  и tab-completion работал.

Followup'ы по уже закрытому шагу — `<NN>-<MM>-followup-<slug>.md`.

## Формат

Строгий — чтобы планирующий Claude мог прочитать взглядом:

```markdown
# Stage <N>, шаг <M>: <human title>

**Дата:** YYYY-MM-DD
**Статус:** ✅ закрыт | ⚠️ частично | ❌ застрял
**Коммиты:** `abc1234..def5678` (диапазон) или список SHA

## Что сделано
- маркированный список конкретики (не «реализовал fetcher», а
  «реализовал _retry_request с backoff min(2**attempt, 60)»)

## Проверки
- pytest tests/test_fetcher.py — 9/9 passed
- ruff check src/ migrations/ tests/ — clean
- mypy src/ — clean
- (любые другие — alembic upgrade/downgrade, ручные curl и т.п.)

## Решения по ходу
- что было неоднозначным и как решил, со ссылкой на docs/ADR если уместно
- если отклонился от плана — здесь объяснение почему

## Открытые вопросы для следующих шагов
- то, что всплыло, но решать не сейчас

## Что дальше
- ссылка на следующий шаг из docs/08-roadmap.md
```

Скелет того же лога — в `_template.md` в этой же директории (см. рядом).

## Что не пишется в session-log

- Информация, которая уже есть в `CONTEXT.md` или `docs/`. Session-log
  — не дубль документации; он фиксирует «как был закрыт конкретный
  шаг», а не описывает архитектуру.
- Полный diff/листинги кода. Достаточно короткой ссылки «см. коммит
  `<sha>`». Файл должен помещаться на экран.
- Эмоции и саморефлексия. Только факты + решения + ссылки.
