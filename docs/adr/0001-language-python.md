# ADR-0001 · Python 3.12 как основной язык

Дата: 2026-05-17  
Статус: принято

## Контекст

Проект — I/O-bound (HTTPS fetch, потоковый парсинг текстовых
форматов, COPY в Postgres). CPU-нагрузка вторична. Команда
предполагается небольшая, единый язык — благо. Целевая аудитория
проекта понимает Python.

## Решение

Python 3.12. `pyproject.toml`, src-layout.

## Следствия

+ Богатая экосистема для парсеров (RPSL, gzip, IPv6), для веба
  (FastAPI), для БД (SQLAlchemy 2 + asyncpg).
+ Одинаковый язык для CLI, ETL и API.
+ Простой онбординг.
− Парсинг 5+ млн RPSL-объектов в чистом Python — десятки секунд.
  Приемлемо для суточного цикла. Если узким местом окажется —
  переписываем горячий цикл на ручной байтовый сканер или
  выносим в Rust-extension (см. альтернатива 2).

## Альтернативы

1. **Go.** Быстрее, отличный концурренси-рантайм. Минусы:
   слабее экосистема под whois/RPSL, придётся писать парсеры
   с нуля; типизация SQLAlchemy/Alembic — лучше в Python.
2. **Rust.** Самый быстрый, но избыточен — мы не упираемся
   в perf. Можно использовать как extension, если в Stage 4
   понадобится ускорять парсеры.
3. **Node / TypeScript.** Большой DX-плюс на API, но слабее
   на CLI/ETL. Тип `bigint` для IPv6 — болезненно.
